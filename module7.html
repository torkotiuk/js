insertAdjacentHTML() - add carefully where needed

If we use HTML parser, mostly we would like to add some markup without deleting old one. It makes with command insertAdjacentHTML():

  • titleRef.insertAdjacentHTML("beforebegin", '<span style="color: red">This is a span </span>' )
    ⇐ put markup before tag <h1></h1>
  • titleRef.insertAdjacentHTML("afterend", '<span style="color: red">This is a span </span>' )
    ⇐ put markup after tag <h1></h1>
  • titleRef.insertAdjacentHTML("afterbegin", '<span style="color: red">This is a span </span>' )
    ⇐ put markup after <h1> before all markup
  • titleRef.insertAdjacentHTML("beforeend", '<span style="color: red">This is a span </span>' )
    ⇐ put markup before </h1> so before all markup

Lesson 14 - Events

GoIt Book

Модуль 7. Занятие 14. События (17.09)

All process of catching events called publishing subscriber.

Browser is publisher, that publish events.
Subscriber - is a developer, form etc. who wants to evoke some code by some event.

Correct names of callback-s:

  • handle*: handleEvent, handleSubjectEvent
  • *Handler: eventHandler, subjectEventHandler
  • on*: onEvent, onSubjectEvent

There are a lot of different event from users, forms etc.

☝ most useful for
- form event: "submit",
- input type="text": "input"
- input type="checkbox": "change"
- button type="radio": "change"

    const form = document.querySelector('.js-register-form');

    form.addEventListener('submit', onFormSubmit);

    function onFormSubmit(event) {
    event.preventDefault();

    const formData = new FormData(event.currentTarget);

    formData.forEach((value, key) => {
    console.log(`${key}: ${value}`);
    });
    }

    JS explanation:

    • After submit form browser restart page by default
      So usually we prevent thing operation by event.preventDefault();
    • How to take values from form. It takes with event.currentTarget, that links form element
      - look at this as an object in console: console.dir(event.currentTarget);
    • console.dir( event.currentTarget.elements )
      - all elements from form saved in this object by adding attribute "name" for them in html:
      name: "email", name: "password", name: "subscription"
    • When we fill the form (in subscription radio choose "basic") and press submit button, in console "event.currentTarget.elements.subscription.value" saved "basic" value
      the same thing for
      - event.currentTarget.elements.email.value
      - event.currentTarget.elements.password.value
    • In practice values from form don't gether by "event.currentTarget.elements"
      Values from form gether with help of the special class "FormData":
      - const formData = new FormData(event.currentTarget);
    • Ofter "formData" sent to server as a data from some form, but for sure we may check it by forEach() method

    To sum up an one element of the form we may take with event.currentTarget.elements, but gathering all data
